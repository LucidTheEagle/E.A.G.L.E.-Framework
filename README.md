## ğŸ¦… The Five Phases

### **E** - ELEVATE
**Diagnose the root problem from altitude (System Design)**

- Identify the **true** problem, not just symptoms
- Map dependencies and bottlenecks
- Define success metrics before building
- Ask: "What system would make this problem obsolete?"

**Output:** Problem specification document

---

### **A** - ARCHITECT
**Map the data flow and logic gates before writing code**

- Design system topology (data flows, state machines, APIs)
- Identify coupling points and failure modes
- Choose the right abstractions
- Create technical specification

**Output:** Architecture diagram + technical spec

---

### **G** - GENERATE
**Rapid deployment of MVP (Minimum Viable Product) infrastructure**

- Build the skeleton first (infrastructure, routing, core logic)
- Implement critical path features only
- Use proven technologies (avoid experimental tools in production)
- Prioritize functionality over aesthetics in v1

**Output:** Functional MVP ready for real-world testing

---

### **L** - LAUNCH
**Strategic deployment into the market feedback loop**

- Deploy to production early (real users > theoretical users)
- Instrument with analytics and logging
- Gather qualitative feedback aggressively
- Identify friction points through observation

**Output:** Live product + user feedback data

---

### **E** - EVOLVE
**Iterative optimization based on real-world friction**

- Let user behavior guide feature priorities
- Refactor based on performance bottlenecks
- Remove unused features ruthlessly
- Compound improvements weekly

**Output:** Mature, battle-tested system
